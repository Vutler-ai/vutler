# Late March 2026 snapshot

Résumé rapide des dix derniers commits pour garder la documentation alignée sur le code (mars 31, 2026).

Update Avril 2026:

- `L4 social scheduling visible in Vutler calendar` est maintenant câblé: quand un agent programme une publication sociale différée, `scheduled_at` est propagé en `due_date`, la tâche sociale crée/maintient un événement dans `tenant_vutler.calendar_events`, et les schedules récurrents projettent aussi leur prochain `next_run_at` dans l'agenda Vutler. Résultat attendu: l'agenda devient enfin la source visible de vérité pour les séquences de posts, au lieu de dépendre uniquement des `scheduled_tasks` et de la queue interne.
- `L5 recurring schedule projection + overdue reconciliation` complète le point précédent: l'API calendrier développe maintenant les occurrences futures des `scheduled_tasks` sur la plage affichée au lieu de ne montrer que demain, et le scheduler ajoute une boucle de réconciliation qui exécute les schedules en retard (`next_run_at <= NOW()`) pour éviter de perdre silencieusement un run après un restart ou un raté de timer.
- `L3 drive lanes + social orchestration scheduling` est maintenant cadré dans le code courant: les espaces Drive agents sont réorganisés sous `Agents/<lane>/<agent>` avec script de reclassement, et le runtime d’agent expose enfin le scheduler réel pour les séquences multi-jours. Le correctif ferme aussi deux trous de persistance/exécution: `due_date` et `reminder_at` sont désormais écrits dans les vraies colonnes `tasks`, et `TaskExecutor` ne réclame plus les tâches planifiées dans le futur.
