## Agent Runtime v2 — Production Architecture (15:30-15:50)
### Problem
- Agent runtime was broken: `_loginAgentRC` method missing, agents couldn't post (401 errors)
- Previous approach: shared password login at startup → tokens expire → fragile

### Solution: PAT-based architecture
1. **New PG table `agents`** — source of truth for agent identity, PATs, config
2. **`agentProvisioning.js`** — full lifecycle: create RC user → generate PAT → store in PG
3. **`agentRuntime.js` v2** — loads PATs from PG at startup, no login needed
4. **Auto-migration** — first startup detects empty `agents` table → migrates from `agent_rc_channels`
5. **Smart routing** — @mention → specific agent, otherwise round-robin (no more all-agents-respond)
6. **Agent skip filter** — agents don't respond to other agents (no infinite loops)

### Key files on VPS

## Agent Auth System — Unified (15:50-16:00)

### Architecture deployed
- **Agent Service Token (AST)** — format: `vutler_ast_{id_short}_{random}`
  - Hash SHA256 stored in PG `agents.service_token_hash`
  - Scopes: chat, email, drive, api (per-agent, stored in PG)
  - 60s cache for high-throughput validation
- **Dual auth**: AST (Bearer header) OR RC PAT passthrough (X-Auth-Token + X-User-Id)
- **Admin ops**: RC admin token required (human admin only)
